**Anthropic Engineering：Claude Code auto mode: a safer way to skip permissions**

Anthropic 发布了 Claude Code 的 auto mode，目标是在「手动审批每一步」和「完全跳过权限」之间找到中间路径。

核心机制是两层防御：输入层用 prompt injection 探针扫描所有工具输出（文件读取、网页抓取、shell 输出等），发现可疑内容后向 agent 注入警告；输出层用 transcript classifier（运行在 Sonnet 4.6 上）对每个 action 评估是否符合用户意图再决定是否放行。

分类器本身是两阶段设计：第一阶段快速单 token 过滤，偏向阻断（FPR 8.5%，FNR 6.6%）；第二阶段只对第一阶段标记的 action 做 chain-of-thought 推理，最终将 FPR 降至 0.4%，FNR 升至 17%。

对真实 overeager action 的 17% 漏检率是诚实的披露——分类器通常能识别危险动作，但在「用户之前给的模糊指令算不算授权」这个判断上容易出错。Anthropic 明确表示，auto mode 不是「仔细手动审批」的替代品，而是针对 `--dangerously-skip-permissions` 用户群体的实质改进。

https://www.anthropic.com/engineering/claude-code-auto-mode

**Training Data：Waymo's Dmitri Dolgov: 20 Million Rides and the Road to Full Autonomy**

**关键洞见：** 做好前 90% 和做好最后那几个 9，是完全不同的工程问题——而正是这个特性，导致了自动驾驶行业反复出现"技术突破带来虚假繁荣"的周期。

Waymo 联合创始人兼 CTO Dmitri Dolgov 毕业于莫斯科物理技术学院，2005 年参加 DARPA 城市挑战赛后一头扎进自动驾驶，近 20 年未曾回头。他亲历了这个赛道从 2016-2017 年第一轮泡沫到大批对手倒下，再到 Waymo 落地成为现实服务的全过程。

在技术架构上，他描述 Waymo Foundation Model 是真正的端到端模型，支撑三根支柱：driver（驾驶决策）、simulator（仿真）和 critic（评估）。他把市场上流行的 world model、world action model、VLA 等概念都映射到这套体系，认为核心差异在于 Waymo 在 end-to-end 基础上叠加了结构化中间表征，以支撑更严苛的闭环评估、闭环训练和强化学习。

> "能力做到 90% 很容易，但你一开始应对前 90% 的方式，决定了你能不能拿到后面那几个 9。"

安全数据上：超过 1.7 亿全自动驾驶英里中，严重碰撞事故是人类驾驶员的 1/13；每 8 天预防一次严重伤害；每周全自动驾驶 400 万英里；目前已在 11 座城市运营，计划今年进入伦敦和东京。

第六代硬件（OHAI 车型）已在内部员工中开放，今年内将向所有用户开放。一个令他意外的能力案例：Waymo 的 LiDAR 能通过扫描公交车底部空间捕捉行人脚步的稀疏反射，提前预测被遮挡行人的出现——这个能力是系统自发涌现的，并非专门设计。
