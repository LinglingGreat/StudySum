### Training Data · Anthropic 的 Boris Cherny：编程的"印刷术时刻"

**Boris Cherny 是 Claude Code 的创始人**，本期是他在 Sequoia 活动上的现场对话，话题覆盖 Claude Code 的诞生、他的极端工作方式、以及对软件行业未来的判断。

**Claude Code 的诞生是一场"预判未来"的赌注**

Boris 2024 年底加入 Anthropic Labs——一个小型内部孵化团队，目标是把模型能力转化为产品。当时编程工具的天花板是"Tab 补全"，他判断可以直接跳到"agent 全写代码"的阶段。最初半年完全没有 PMF，直到 Opus 4 发布才开始指数级增长，此后每次模型迭代都会再次拐点上扬。

> 核心策略：为下一代模型而建，而不是当前模型。

**他本人的工作方式已经极端到"失去参照系"**

- 代码 100% 由 AI 写，自己一行不写
- 每天提交几十个 PR，最多的一天做了 150 个
- 主要用**手机**上的 Claude app 工作
- 同时运行数百个 session，每晚有数千个 agent 在做深度工作
- 用 `/loop`（定时循环 agent）管理：PR 自动 rebase、CI 修复、每 30 分钟从 Twitter 拉反馈并聚类……"Loop 就是未来。"

**SaaS 末日论？他的判断是反直觉的**

不是"所有 SaaS 都会死"，而是"护城河的相对重要性"在重新排序：

- 变弱的护城河：切换成本（模型能帮你迁移）、流程壁垒（模型能爬山式优化任何流程）
- 不变的护城河：网络效应、规模经济、独占资源

同时，AI 让小团队可以正面竞争大公司——大公司有组织惯性，小团队原生 AI 没有包袱。**现在是创业最好的时机。**

**编程会成为像识字一样的普世技能**

类比 15 世纪印刷机：印刷机发明前欧洲只有 10% 人口识字，500 年后识字率达 70%。Boris 认为软件编程正在经历同样的民主化，速度会快得多。**最懂某个领域的人，未来会是最好的软件开发者**——因为懂领域才是难点，写代码只是工具。

**Anthropic 内部的组织实验**

Claude Code 团队里，工程师、产品经理、设计师、数据科学家、财务、用研——**所有人都写代码**。各自 Claude 实例全天通过 Slack 互相通信，协调未知信息。没有任何手写 SQL，全部由模型生成。Boris 认为这才是真正的领先优势——不是技术，而是**组织结构**。
