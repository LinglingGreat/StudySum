### Anthropic Engineering：将 Managed Agents 的"大脑"与"双手"分离

**核心结论：** 把 Claude 的推理层（大脑）与执行环境（双手）物理解耦，是构建可靠、可扩展长任务 agent 系统的关键架构决策。

Anthropic 工程博客这篇文章详细拆解了 Managed Agents 的架构演进。早期版本将 Claude 的 harness 和 sandbox 放在同一个容器里，结果是任何一处故障都会拖垮整个 session，调试困难，也难以对接客户自己的 VPC 环境。

重构的核心是引入三层解耦：

**Session（会话日志）** 独立存储在容器外，任何 harness 崩溃后都可以通过 `wake(sessionId)` 重启并从上次断点继续，不丢失状态。

**Harness（大脑）** 不再驻留在容器内，通过统一接口 `execute(name, input) → string` 调用 sandbox 和任何工具，包括 MCP server 和自定义工具。这意味着容器变成了"牲口"而非"宠物"——失败了直接换一个，不用人工干预。

**Sandbox（双手）** 只在真正需要时才由 harness 通过工具调用按需创建，而不是每个 session 都提前开好容器。这一改变让 p50 首字延迟降低了约 60%，p95 首字延迟降低超过 90%。

安全边界方面，凭据永远不会出现在 sandbox 里：Git token 在 clone 阶段就被 wire 进本地 remote，MCP OAuth token 存在 vault 外部，通过 proxy 代理调用，Claude 本身从来不持有任何凭据。

文章最后的类比很有意思：操作系统几十年来靠"虚拟化硬件为抽象层"来适应未来的软件，Managed Agents 做的是同样的事情，把 harness、sandbox、session 抽象为稳定接口，让底层实现可以自由演化。

https://www.anthropic.com/engineering/managed-agents

### No Priors：Baseten CEO Tuhin Srivastava 谈 AI 推理市场的供需紧张、定制模型与推理云的未来

**核心结论：** AI 推理市场的真正大爆发还在前面，99% 的推理量目前仍来自 AI 原生应用公司，企业级采购才刚刚开始转向。

Baseten 是一家 AI 推理云公司，过去一年增长了 30 倍，今年预计营收突破十亿美元。CEO Tuhin Srivastava 在这期播客里分享了他对推理市场当前状态和未来走向的判断，信息密度很高。

**应用层会消亡吗？** Srivastava 给出了一个有说服力的反驳：前沿模型公司无法获取垂直领域的用户信号。他举了 AI 医疗转录公司 Abridge 为例，医生在 EMR 系统里对 AI 生成内容的每一次修改，都是只有 Abridge 才能拿到的反馈数据。这类数据无法被 OpenAI 或 Anthropic 直接获取，而这正是应用层建立护城河的真正来源。

> "The majority of the market hasn't come online and added AI into the system yet."

**开源模型的趋势：** 客户基本都从能力优先出发，先找最好的模型，再优化成本。Deepseek、Moonshot、Canopy、Orpheus 这些来自不同背景的开源模型，都被 Baseten 的客户使用，选择标准是"谁在当下这个任务上最强"。

**中国模型的地缘政治问题：** Srivastava 的态度是：他没有见过真实的安全后门证据（除了早期个别案例），但他也认为美国保持自己的模型研发能力在战略上是必要的。

**定制模型的时机：** 他的判断是，企业在用过闭源 API 之后，会逐步意识到可以做什么，然后才会走向定制模型。这条路目前还在早期，大量采用还在前面。
