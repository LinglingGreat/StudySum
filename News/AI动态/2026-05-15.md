## OFFICIAL BLOGS

**Claude Blog：New in Claude Managed Agents: dreaming, outcomes, and multiagent orchestration**

Anthropic 为 Claude Managed Agents 发布了三项重要更新：

**Dreaming（记忆自我进化）**：一项定期执行的后台进程，会回顾历史 session、提炼规律、更新 memory store，让 agent 随时间自我改进。开发者可选择全自动更新或逐次审核。Harvey 在内测中任务完成率提升约 6 倍。

**Outcomes（目标导向评分）**：开发者写好成功标准（rubric），独立 grader 在独立 context 中评估输出、不受 agent 推理链干扰，不达标则 agent 自动重跑。文档生成任务成功率最高提升 10 个百分点（docx +8.4%，pptx +10.1%）。也支持 webhook 通知：定义 outcome 后让 agent 跑，完成时收到通知。

**Multiagent Orchestration（多 agent 协作）**：主 agent 拆解任务后，将子任务分配给各自有独立 prompt 和工具的专家 agent 并行执行，共用文件系统，结果汇总回主 agent。Netflix 平台团队用此方式并行分析数百次构建日志，仅汇报跨多次构建重复出现的问题模式。所有步骤在 Claude Console 中可全程追溯。

## PODCASTS

**Training Data：Suno's Mikey Shulman: Everyone Can Make Music Now**

**核心洞察：** AI 音乐的意义不在于"更好的 Spotify"，而在于把创作本身变成了一种娱乐行为，让全人类第一次都能成为音乐创作者。

Mikey Shulman 是 Suno 创始人兼 CEO，哈佛物理学博士出身（量子计算方向），最终带着团队建起了全球最大的 AI 音乐公司。他的核心判断是：在 Suno 出现之前，80 亿人里只有极少数人在"做音乐"，其余人都是被动消费者。Suno 改变了这一点——每天 90% 的用户会主动创作，而他们创作的目的不是为了发布或传播，创作本身就令人满足。

Suno 在技术路线上做了一个关键决策：完全不向模型灌输乐理知识。

> "如果告诉模型西方音乐只有 12 个音，它就永远被困在那 12 个音里。我们选择把音乐视为纯粹的声波序列，让模型从零学习。"

这个决策催生了真正新颖的声音——trap 混搭西塔琴、乡村音乐加 808 鼓机，甚至微分音音乐。

Shulman 还打破了一个常见误解：AI 音乐不是 scale 问题。Suno 的模型相对较小，因为音乐没有可量化的基准测试——你和我对"好音乐"的定义就不一样。真正的护城河来自大规模用户偏好数据，这让他们能做 LLM 领域做不到的纯偏好对齐，也是 V5 实现质量跃升的核心原因。

商业合作上，Suno 已与 Warner Music 达成合作，目标不是取代唱片公司，而是共同创造此前从未存在过的产品形态：让粉丝通过音乐与艺人深度互动。Shulman 预测，未来 12 个月内将出现真正的互动演唱会——观众实时与艺人共同创作歌曲。他认为，音乐在过去 30 年里越来越变成"背景"，而 AI 可以重新让音乐回到社会性、参与性的核心位置。

Suno 目前年化收入已达 3 亿美元，但 Shulman 认为还非常早期，接下来的重点是让音乐创作更加社交化（异步共创、模板分享）以及让用户能把自己的声音融入作品。
